Northbound API

northbound API

class netaigym.client.northbound_api

The northbound_api connects the client to the server, configure the environment parameters, communicate network stats and policy between client and netaigym envrionment.

Methods

Method
netaigym.client.northbound_api.connect(self)

Connect to the netaigym environment via netaigym server.

Method
netaigym.client.northbound_api.send(self, policy: JSON)

Send the policy to the netaigym envrionment.

Method
netaigym.client.northbound_api.recv(self) -> network_stats: list[pandas.DataFrame]

Receive the network stats from the netaigym environment.

Attributes

Network Environment Simulation Parameters

Prior to initiating a simulation for a network environment, the network environment parameters, denoted as "gmasim_config", can be customized using JSON files. For instance, the nqos_split_config.json (opens in a new tab) file is utilized to configure the nqos_split environment. It's important to note that certain parameters may only be relevant to specific subsets of environments. For example, the parameter "slice_list" is exclusively applicable to the Cellular RAN Slicing environment.

ParameterDescription

random_runchange the random seed for this simulation
downlinkset to true to simulate downlink data flow, set to false to simulate uplink data flow
max_wait_time_for_action_msthe max time the netaisim worker will wait for an action. set to -1 will cap the wait time to 100 seconds.
enb_locationsx, y and z locations of the base station, we support 1 base station only
ap_locationsx, y and z location of the Wi-Fi access point, add or remove element in this list to increase or reduce AP number. We support no AP as well
num_usersthe number of users
slice_listnetwork slicing environment only, resouce block group (rbg) size maybe 1, 2, 3 or 4, it depends on the resource block num, see table 7.1.6.1-1 of 36.213
user_left_right_speed_m/suser speed (m/s). Users only move to left or right. When a user reaches the doundry, it reverse the direction
user_location_rangeinitially, users will be randomly deployed within this x, y range. if user_left_right_speed_m > 0, the user will move left and right within this boundary
app_and_measurement_start_time_mswhen the application starts traffic and send measurement to RL agent

min_udp_rate_per_user_mbpsif "transport_protocol" is "udp", this para controls the min sending rate
max_udp_rate_per_user_mbpsif "transport_protocol" is "udp", this para controls the max sending rate

GMA:measurement_interval_msduration of a measurement interval for GMA

GMA:respond_action_after_measurementwhether to response action after receiving measurement, do not change it

Network Stats

At the end of each measurement interval, the netaigym API communicates the following networks stats measurement (opens in a new tab) to the client. The client will use the adapter to transform these measurements into observations and rewards.

MeasurementUnitDescription
max_ratembpsLTE/Wi-Fi link capacity measured by each user
loadmbpsinput traffic throughput measured by each user
ratembpsoutput traffic throughput measured by each user, including LTE, Wi-Fi, and ALL
qos_ratembpsoutput traffic throughput that meet the QoS requirement measured by each user, including LTE, Wi-Fi, and ALL
owdmsone-way delay measured by each user, including LTE, Wi-Fi, and ALL
tsutraffic split ratio measured by each user, including LTE, Wi-Fi
ap_idaccess-point/cell ID measured by each user, including LTE and Wi-Fi
slice_idLTE slice ID measured by each user
rb_usage%LTE resource block usage percentage (%) measured by each user
delay_violation%packet One-way delay violation percentage (%) measured by each user
Network Environment/Policy

When the agent returns a action, the netaigym client utilizes the adapter to convert the action to policy, which will be applied to the netaigym environment.

EnvironmentPolicyDescription
nqos_split: Multi-Access (MX) Traffic SplittingTraffic SplittingThe agent updates traffic split ratio over Wi-Fi and LTE for each user
qos_steer: QoS-aware MX Traffic SteeringTraffic SteeringThe agent steers traffic over Wi-Fi or LTE for each user
network_slicing: Cellular RAN SlicingResource AllocationThe agent updates LTE resource block allocation ratio for a slice
